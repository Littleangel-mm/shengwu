# 用户使用指南

## 适用范围

本指南覆盖当前系统的文献解析、证据检索、术语、字段方案、抽取审核、数据集、
模型、预测优化和报告流程。界面显示“完成”只表示系统任务完成，不代表客户业务验收通过。

## 推荐操作流程

1. 登录后进入所属组织和项目；无写权限时联系项目所有者调整角色。
2. 上传 PDF、DOCX、XLSX、TXT、Markdown 或安全 ZIP，等待解析任务结束。
3. 在文献详情检查正文、表格、图片、页码、坐标和 OCR 状态。`partial` 表示结果不完整，
   应先处理 OCR 或版式问题，不应直接当作完整证据。
4. 创建检索任务，确认或排除命中；审核术语及别名后再用于字段方案。
5. 创建字段方案，配置字段类型、单位、必填、建模和评分属性；确认后冻结版本。
6. 运行抽取，逐条核对原始值、标准值、处理组、时间点、实验条件和证据位置。
7. 构建数据集，处理缺失值和疑似值。冻结后数据和证据不可修改；有变更时克隆新版本。
8. 使用冻结数据集训练模型，查看测试与交叉验证指标，人工选择模型后再预测或优化。
9. 生成并下载 Excel 数据或 Word 研究报告，抽查证据附录和截断提示。

## 审核原则

- 系统不会从未印刷的趋势线推测数值；此类图表应保留为人工处理项。
- 扫描文档必须检查 OCR 置信度和证据坐标。
- 数值审核必须同时检查单位、标准值和客户确认的容差。
- 翻译依赖有效的服务密钥与额度；机器结果应由客户指定双语审阅人盲评。
- 删除、合并、冻结和模型选择等重要操作应由有权限人员执行并保留审计记录。

## 金标准与 UAT

开发人员可用合成 JSON 验证评估工具，但合成结果不得写成客户真实准确率。正式测试前，
客户须签字确认样本版本、SHA-256 清单、指标阈值、容差、权重、严重缺陷和一票否决项。
客户应按真实业务流程执行 UAT，并在 `UAT_ACCEPTANCE_REPORT.md` 填写结果和签字。

## 常见状态处理

- 任务长时间停留在等待：确认 Worker 正在运行，再由运维人员检查任务日志和心跳。
- 文档为 `partial`：检查 OCR 就绪状态、原文件质量和失败页，不要重复上传覆盖原版本。
- 无法修改：确认角色权限以及字段方案、数据集是否已冻结。
- 翻译失败：检查服务密钥、额度、网络、限流与截断信息，不要在日志中粘贴密钥。
- 证据不一致：标疑或排除记录，保留问题说明，再由业务审核人决定。
