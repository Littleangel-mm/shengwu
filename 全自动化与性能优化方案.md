# 全自动化与性能优化方案（fr-v1）

> 方向（已与用户确认）：**保留"自动发现"路线**，不改成让客户手写领域包；
> 目标是让它**真正像全自动**（少填/最好零填）+ **解决 7 分钟/篇的性能问题**。
> 本文先定方案，评审通过后再动手写代码。

---

## 1. 问题定性（有代码依据）

### 1.1 性能：7 分钟/篇
- `parser.py` 对**无文字层的页**逐页渲染成图 + 跑 PaddleOCR 独立 venv 子进程（第 371–394 行）。扫描版论文=每页都 OCR → 分钟级。
- 即使有文字层，仍额外做：图片抽取、图片 OCR、表格单元格检测、每次子进程加载模型。
- 对比客户原型：只用 `pypdf` 读文字层、PyMuPDF 兜底，**无 OCR、无图表抽取** → 秒级。
- 客户测试集"几百篇"，当前吞吐量根本扛不住。

### 1.2 交互：要填太多东西
当前一个项目从上传到报告，用户要**手动触发并逐个确认**约 10+ 个阶段：
```
上传 → 解析 → 跑检索 → 跑字段发现 → 逐个确认候选字段 → 建 Schema → 冻结 Schema
→ 跑抽取 → 逐条复核记录 → 确认单位换算 → 建数据集 → 冻结 → 建模 → 优化 → 报告
```
每个阶段都是独立按钮 + 独立确认页。对非技术客户过于繁琐。

---

## 2. 目标（本次要达到的体感）

1. **一键直达**：上传文献后点一个"开始自动处理"，系统自动跑完 解析→发现→建Schema→抽取→建数据集草稿，中途不打断。
2. **只在最后复核少数项**：用户只看"低置信度/异常/单位冲突"的少量条目，其余自动采纳。
3. **秒级解析**：常规电子版 PDF 秒级完成；OCR 仅对扫描件、且用户手动开启。
4. **高级选项折叠**：所有参数（置信度阈值、页码范围、模型选择等）默认给合理值，折叠进"高级设置"。

---

## 3. 方案设计

### 阶段 P1 — 解析性能（先做，硬伤）
- **文字层优先**：默认只走 `page.get_text("text")`；整篇无文字层时才判定为"疑似扫描件"。
- **OCR 改为按需**：新增文档级 `needs_ocr` 标记；OCR 默认关闭，仅当用户对"疑似扫描件"手动点"启用 OCR"时才跑。
- **图片/表格抽取降级为可选**：默认关闭重型图表抽取（图片 OCR、单元格检测），放到"深度解析"可选项，或后台异步补跑，不阻塞主流程。
- **PaddleOCR 常驻/批处理**：若开 OCR，一次子进程批量处理整批图片，避免逐页冷启动。
- 验收：一篇 20 页电子版 PDF ≤ 10 秒；批量 100 篇不因单篇卡死。

### 阶段 P2 — 一键自动流水线（核心，解决"填太多"）
- 新增编排服务 `AutoPipelineService` + 一个后台 job `auto_pipeline`，顺序执行：
  1. 解析（P1 后的快速解析）
  2. 全文检索建索引（喂给发现）
  3. 字段发现 `discover_fields`（已有）
  4. **自动采纳高置信候选**：occurrence/命中数超阈值的候选自动进 Schema，其余进"待复核"
  5. 自动生成并冻结一版 Schema（可回退）
  6. 抽取 `execute`（已有）
  7. **自动分流**：高置信记录自动 confirmed，低置信/超范围/单位冲突标 doubtful 待人工
  8. 生成 dataset 草稿版本
- 前端：一个"开始自动处理"按钮 + 实时进度（复用已有 job 日志），完成后直接跳"复核少数项"。

### 阶段 P3 — 复核体验精简
- **复核收敛页**：只列 doubtful/冲突/低置信，支持"全部采纳/批量处理"，不再逐条强制。
- 高级参数（阈值、页码、上下文宽度、模型勾选）默认隐藏，放"高级设置"折叠区。
- 空/少数据阶段给占位提示，不报错、不中断。

### 阶段 P4 — 向导式引导 + 回归
- 首次进入项目给 3 步向导：①上传 ②一键处理 ③看结果/下报告。
- 用雪茄样本 + 一篇非雪茄生物文献各跑一遍，验证"上传→一键→报告"全程无需手动配置字段。

## 4. 任务拆解与排期（建议）

| 阶段 | 任务 | 关键改动文件 | 预估 |
|------|------|--------------|------|
| P1 | 文字层优先 + OCR 按需 + 图表降级 | `parser.py`、`config.py`、`document.py`、前端解析入口 | 1–1.5 天 |
| P2 | AutoPipeline 编排 + auto_pipeline job + 自动采纳阈值 | 新增 `auto_pipeline.py`、`job.py`、`term.py`、`extraction.py`、`dataset.py` | 2–3 天 |
| P3 | 复核收敛页 + 高级参数折叠 | 前端 `App.tsx`、`api.ts`、`index.css` | 1–1.5 天 |
| P4 | 向导 + 跨领域回归验证 + 测试 | 前端向导、`tests/` | 1 天 |

> 每阶段结束跑：后端 ruff/mypy/pytest + 前端 tsc/build/lint。

## 5. 验收标准

- [ ] 电子版 PDF 秒级解析；扫描件才提示"可开启 OCR"。
- [ ] 上传后**一键**跑完 解析→发现→Schema→抽取→数据集草稿，无需手动逐步点。
- [ ] 常规流程用户**零字段配置**即可拿到抽取结果与报告；仅需复核少量不确定项。
- [ ] 高级参数全部有默认值并折叠。
- [ ] 雪茄 + 另一生物领域文献均可跑通"上传→一键→报告"。

## 6. 待你确认的取舍点

1. **自动采纳阈值**：高置信候选自动进 Schema 的阈值定多少（例如出现≥3次或命中字段数≥N）？先给默认、可在高级设置调。
2. **图表抽取**：默认关闭、需要时再"深度解析"，可以接受吗？（这是提速关键之一）
3. **OCR**：默认关闭、仅扫描件手动开，可以接受吗？
