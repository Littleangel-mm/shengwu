# 生产上线检查清单

> 本清单是上线前模板。未填写证据和责任人的项目不得标记为通过。

## 配置与安全

- [ ] 生产配置与开发环境隔离，`.env`、密钥和凭据未进入仓库或日志。
- [ ] `APP_SECRET` 为持久随机值，`ALLOW_ACTOR_HEADER=false`。
- [ ] 数据库、存储、服务账户和网络权限遵循最小权限。
- [ ] TLS、允许来源、反向代理、上传大小和超时符合部署方案。
- [ ] 平台管理员、组织管理员、项目角色及离职账户已复核。
- [ ] 依赖审计、凭据扫描和绝对路径扫描通过，结果已归档。

## 数据库与存储

- [ ] 生产数据库版本与 Alembic revision 已记录。
- [ ] 待执行迁移已评审，维护窗口和负责人已确认。
- [ ] 上线前备份完成，校验值及异地副本可用。
- [ ] 独立环境恢复演练通过，RPO/RTO 和演练证据已记录。
- [ ] 数据库和对象/文件存储容量、权限、保留与清理策略已配置。

## 服务与可观测性

- [ ] API 和 Worker 由服务管理器托管，重启策略和日志轮转生效。
- [ ] 数据库、API、Worker、OCR 健康检查均通过。
- [ ] 错误率、延迟、磁盘、连接数、任务积压、心跳和备份告警已验证。
- [ ] 日志脱敏，告警不包含令牌、密码或客户文献正文。
- [ ] DeepSeek 密钥、额度、限流和失败处理已由责任方验证；不使用时已禁用。

## 构建与验证

- [ ] Ruff、Mypy、Pytest 通过，命令、日期、版本和完整输出已归档。
- [ ] 生产制品可复现，版本号和 SHA-256 已记录。
- [ ] 数据库迁移、健康检查、上传解析和只读查询冒烟测试通过。
- [ ] 备份、恢复、故障升级和回退 runbook 已演练。
- [ ] 已知限制、遗留风险、责任人和处理日期已获批准。
