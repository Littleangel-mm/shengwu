# 新手使用指南（零基础）

本指南写给**不懂编程、也不了解机器学习**的研究人员。跟着做，即可完成
“上传文献 → 得到可用于研究的结构化数据、模型与报告”的全过程。你**不需要**
提前定义任何字段或规则——系统会自动从你的文献里发现候选字段和关键词，
你只需**看一眼、点确认**。

> 重要提醒：界面上显示“完成”只代表系统的这一步任务跑完了，**不等于**结果
> 已经通过你的业务验收。每一步都建议你抽查一下，尤其是数值和证据。

---

## 一、准备工作

1. 用账号登录，进入你所在的**组织**和**项目**。
2. 如果发现只能查看、不能操作，说明你的角色是“只读”，请联系项目所有者
   把你设为“编辑者”或“所有者”。

---

## 二、五步走完整流程

### 第 1 步：上传文献

- 进入「文献管理」，把文件拖进上传区，或点击选择。
- 支持格式：**PDF、DOCX、XLSX、CSV、TXT、Markdown、ZIP**（打包上传）。
  - Excel 旧版 `.xls` 请先另存为 `.xlsx` 再上传。
  - CSV / Excel / 规整表格会被自动识别成表格数据。
- 单个文件最大 **1 GB**。
- 上传后系统会自动开始“解析”。你可以点每个任务的「日志」查看进度。

**建议抽查**：解析完成后打开「文献详情」，看看正文、表格、页码是否正确。
如果状态是 `partial（不完整）`，通常是扫描件需要文字识别（OCR）或版式复杂，
先处理好再往下走，不要把不完整的结果当成完整证据。

### 第 2 步：让系统自动发现字段（零配置的核心）

- 进入「术语与字段」，点击**自动发现字段**。
- 系统会扫描你所有文献的正文、表格表头和图表标题，自动找出候选字段，例如
  “发酵温度、含水率、感官评分”等，并附上：出现频次、跨文献篇数、常见单位、
  几条**原文示例**、以及可能的同义写法。
- 稍等片刻（任务完成后），页面会列出这些**候选字段**。

> 它是怎么找出来的？简单说：一是找“短语 + 数字 + 单位”的写法，二是抓表格
> 表头，三是统计高频专业词。可选地，再由大模型帮忙把名字规范化、归类。
> 详细原理见《自动发现机制说明》。**数值永远来自原文，不会由大模型编造。**

### 第 3 步：确认字段，冻结字段方案

这一步就是“看一眼、点确认”，你可以：

- **保留/删除**：把不需要的候选去掉。
- **合并同义**：系统会给出“同义建议”（例如“温度”和“fermentation temperature”
  可能是一回事），你确认后点合并。
- **微调**：修改显示名、单位、字段角色（特征 / 目标 / 标识 / 元数据）。

确认满意后，点**从候选生成字段方案**，再点**冻结**。冻结后这套字段就固定了，
保证后续抽取和数据可追溯、可复现。

> 关键词/术语也可以类似地“自动发现 + 确认”，用于检索和辅助字段整理。

### 第 4 步：抽取数据并核对

- 用冻结后的字段方案，创建并运行**数据抽取**。
- 系统会按字段名和同义词，从原文里用规则抽取数值，并保留每个值的
  **原文证据（页码、坐标、原句）**。
- 逐条核对：原始值、单位、处理组、时间点、实验条件是否正确。有疑问的可以
  标“疑似”或“排除”，并写明原因。
- 核对无误后，**构建数据集**并**冻结**。冻结后的数据与证据不可修改；若要改，
  克隆一个新版本再改。

### 第 5 步：建模、优化与出报告

- 用冻结数据集**训练模型**，查看测试指标与交叉验证结果，**由你人工选择**
  要采用的模型。
- 需要找“最优实验条件”时，运行**参数优化**，得到推荐条件、不确定度，以及
  是否落在“可靠范围（适用域）”内，并附**验证实验方案**。
- 最后**生成 Word 研究报告**并下载。报告自动包含：摘要、研究目标、研究流程图、
  （如启用）PRISMA 文献筛选流程、数据与字段说明、建模设计与结果、优化推荐、
  验证方案、证据索引、可复现性、开源软件版本、发表建议等章节。

## 三、可选功能

### 系统综述（PRISMA）开关

如果你的研究是**系统综述 / 荟萃分析**，可在「项目概览」打开
“启用 PRISMA 流程模块”。开启后侧栏出现「系统综述」页，可录入文献识别、
去重、筛选、纳入各阶段的数量，系统会画出 PRISMA 2020 流程图并写进报告。
纯实验数据类项目可以不开。

### 报告血缘（来源链路）

报告生成后，在报告列表点「血缘」，可以看到这份报告是从哪些**原始文件**、
经过哪一步步（检索 → 字段方案 → 抽取 → 数据集 → 模型 → 优化 → 报告）得到的，
以及每一环的**哈希值**，用于证明数据可追溯、可复现。

## 四、常见问题

| 现象 | 处理办法 |
|------|----------|
| 任务一直“等待中” | 确认后台 Worker 正在运行，再请运维查看任务日志与心跳 |
| 文献显示 `partial` | 检查 OCR 是否就绪、原文件是否清晰、有无失败页；不要重复覆盖上传 |
| 无法修改 | 检查你的角色权限，以及字段方案 / 数据集是否已冻结 |
| 抽取数值不对 | 对该记录标“疑似”或“排除”，写明原因，交业务审核人判断 |
| 翻译失败 | 检查翻译服务密钥、额度、网络；不要把密钥贴进日志 |
