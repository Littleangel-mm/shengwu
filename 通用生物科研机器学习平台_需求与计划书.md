# 通用生物科研机器学习平台 — 需求与建设计划书

> 版本：v1.0　｜　状态：需求整理稿　｜　编写目的：作为平台重构的需求基线与验收依据
>
> 说明：本文只负责"需求与计划"整理，不负责具体架构落地（架构在另一工程目录实现）。

---

## 1. 项目背景

现有原型是一个针对「雪茄烟叶发酵工艺参数预测」的本地 Streamlit 应用，已经跑通了一条完整链路：

```
文献/数据上传 → 自动抽取 → 文献筛选 → 单位换算与质量评分 → 人工校对 → 机器学习建模 → 工艺参数优化推荐 → 学术 Word 报告
```

原型的价值不在"雪茄"这个具体领域，而在这条**"从非结构化文献到可追溯数据集，再到机器学习与参数优化，最后到可发表报告"**的科研方法链路。

**客户新需求**：把它升级为**通用平台**——不局限于雪茄发酵，任何生物相关项目（微生物发酵、食品发酵、酶催化、植物/作物性状、生物反应器、环境微生物、代谢/发酵优化等）都能使用。

---

## 2. 建设目标

1. **领域可配置**：领域相关的一切（字段、抽取关键词、单位体系、质量规则、目标/特征、优化变量、报告模板）都由**配置**驱动，而非写死在代码里。切换领域 = 换一份配置，不改内核代码。
2. **通用科研链路**：保留并强化"文献→数据集→建模→优化→报告"这条方法学链路，使其对任意生物领域的表格/文献数据都适用。
3. **可追溯、可复现**：任意一个数字都能回溯到来源；相同输入+配置+随机种子可复现结果。
4. **低门槛**：不懂编程的科研人员也能通过界面完成全流程；同时给专业用户开放高级参数。
5. **可扩展**：抽取器、模型、优化算法、报告模板都应可插拔扩展。

---

## 3. 目标用户与典型场景

| 用户 | 场景 |
|------|------|
| 研究生 / 科研人员 | 汇总文献数据、建模、写论文方法与结果章节 |
| 工艺/研发工程师 | 用历史实验数据建模，推荐最优工艺参数 |
| 实验室团队 | 建立可追溯的领域数据集，团队复用 |

**跨领域示例（说明"通用"到什么程度）**
- 雪茄/烟叶发酵：温度/含水率/时间 → 感官评分、化学指标
- 食品发酵（酱油/酸奶/泡菜）：菌种/温度/盐度/时间 → 风味/酸度/微生物指标
- 微生物发酵/生物反应器：接种量/溶氧/pH/转速/补料 → 产物浓度/得率
- 酶工程：温度/pH/底物浓度 → 酶活/转化率
- 植物/作物：基因型/环境/栽培措施 → 产量/品质性状

> 这些场景的共性：**一组可控输入变量 + 一组可测结果变量 + 需要从文献或实验表格汇总数据 + 需要建模并反推最优输入**。平台就是把这个共性做成通用工具。

## 4. 核心设计理念：领域无关内核 + 领域配置包

```
┌─────────────────────────────────────────────┐
│              领域无关内核 (Core)               │
│  接入 / 抽取引擎 / 筛选 / 质量 / 建模 / 优化 / 报告   │
│  —— 只依赖"数据契约"和"配置"，不含任何领域知识 ——      │
└───────────────────────┬─────────────────────┘
                        │ 读取
              ┌─────────┴──────────┐
              │    领域配置包 (Domain Pack)     │
              │  字段Schema / 抽取关键词 / 单位规则 │
              │  质量规则 / 目标特征 / 优化变量     │
              │  报告模板 / 领域文案               │
              └────────────────────────────────┘
   例：雪茄发酵包、食品发酵包、酶工程包……（用户可新建/编辑）
```

**关键点**：现有原型里所有硬编码的领域知识（`CANONICAL_COLUMNS` 57 个字段、`KEYWORD_MAP` 关键词正则、单位换算、质量合理范围、质量指数权重、报告中文文案）都要**外置为领域配置包**。

## 5. 系统范围与总体流程

平台功能范围（9 大模块）：

```
[0] 领域配置中心 ──> 定义/选择领域包（本次新增的核心能力）
 │
[1] 数据接入 ──> [2] 智能抽取 ──> [3] 文献筛选 ──> [4] 标准化与质量
                                                      │
[8] 报告导出 <── [7] 优化与推荐 <── [6] 建模 <── [5] 数据校对
                                                      │
[9] 项目/运行管理与可追溯（横切，贯穿全程）
```

## 6. 功能需求（分模块）

### 模块 0｜领域配置中心（本次新增，最关键）

**功能**：让平台脱离"雪茄"绑定，可为任意生物领域定义/管理配置包。

- 内置若干示例领域包（至少含"雪茄发酵"作为迁移样例）
- 用户可**新建/复制/编辑/导入/导出**领域包（建议 YAML/JSON）
- 一个领域包定义：
  - **字段 Schema**：字段名、中文名、类型（数值/类别/文本）、单位、角色（可作输入/目标/追溯/元数据）、合理取值范围
  - **抽取规则**：每个字段的关键词与正则模式、单位捕获规则
  - **单位体系**：可换算单位对及换算公式
  - **质量规则**：完整度分区、合理范围校验
  - **派生特征定义**：可配置的派生公式（如比值、乘积、积温类）
  - **优化变量**：哪些字段是"可搜索的可控输入"、默认搜索范围
  - **报告模板**：章节结构与领域文案
- 项目创建时选择一个领域包；后续所有模块读取该配置

**验收标准**
- [ ] 不改动内核代码，仅新增一个领域包，即可让平台服务一个全新生物领域
- [ ] 至少提供 2 个可用领域包示例（雪茄发酵 + 另一个生物领域）
- [ ] 领域包可在界面查看/编辑并校验合法性（字段引用、单位、正则语法）
- [ ] 领域包可导出为文件、可从文件导入
- [ ] 缺失或非法配置有清晰报错，不静默失败

---

### 模块 1｜数据接入

**功能**：接收文献与数据文件并解析。

- 输入：PDF / CSV / Excel(xlsx/xls) / ZIP，多文件、大文件（≥1GB）
- 输出：页文本表 + 结构化表 + 输入文件清单（含 SHA256）

**验收标准**
- [ ] 多文件上传；ZIP 安全解压（拒绝路径穿越，仅放行白名单类型）
- [ ] 每个输入文件记录 SHA256、大小、类型到 lineage
- [ ] 大文件流式分块读写，不因一次性载入内存而崩溃
- [ ] 非法类型明确报错
- [ ] 每次运行生成独立时间戳目录，历史不被覆盖
- [ ] （通用化）结构化表的列**按领域包 Schema 对齐**，未知列保留但标注

---

### 模块 2｜智能抽取

**功能**：从 PDF 文本按领域配置抽取字段，生成候选记录。

- 输入：页文本 + 领域包抽取规则 + 抽取参数
- 输出：候选记录表（含置信度、原文片段）

**验收标准**
- [ ] PDF 双引擎（pypdf 主 + PyMuPDF 兜底），可手选
- [ ] 抽取字段、关键词、正则**全部来自领域包**（不写死雪茄字段）
- [ ] 可配置：页码范围、最小字段数、置信度基数/增量、上下文宽度、最小置信度
- [ ] 支持临时自定义字段+关键词
- [ ] 置信度 = `min(base + per_field × 命中字段数, 0.95)`，界面明确标注"启发式分数、非统计概率"
- [ ] 每条候选记录保留 source_file / page / original_text / confidence
- [ ] 能推断文档标题（含中文学位论文场景）
- [ ] 抽取过程写入 audit_log
- [ ] （质量修复）正则支持单位捕获与边界约束，减少误匹配（如 pH 误撞）

---

### 模块 3｜文献筛选（PRISMA）

**功能**：记录文献纳入/排除决定并统计。

**验收标准**
- [ ] 可编辑筛选表：标题/作者/年份/期刊/DOI/数据库/检索式/阶段/决定/排除原因/备注
- [ ] 决定=include/exclude/uncertain；阶段与排除原因为可配置下拉项
- [ ] 自动统计 PRISMA 计数并可保存 CSV
- [ ] 有候选数据时可自动初始化筛选表
- [ ] （通用化）排除原因清单可按领域包定制

> 备注：PRISMA 属"文献综述型"研究方法。若某些领域用不上（纯实验数据），该模块应**可跳过/可隐藏**。

---

### 模块 4｜标准化与数据质量

**功能**：统一单位、检测冲突、按领域规则打质量分。

**验收标准**
- [ ] 单位换算规则来自领域包，记录换算日志
- [ ] 单位冲突报告：列出每字段出现的不同单位并标记冲突
- [ ] 质量评分按领域包定义的分区与完整度计算 0–100
- [ ] 合理范围校验来自领域包（雪茄示例：温度15–70℃、含水率5–60%、时间0–365天）
- [ ] 质量报告、冲突报告、换算日志分别落盘

---

### 模块 5｜数据校对

**功能**：网页表格逐条人工核对/修改，形成最终建模数据集。

**验收标准**
- [ ] 表格可增删改行，按领域 Schema 列顺序展示
- [ ] 可保存 curated 数据集
- [ ] 实时预览领域包定义的派生变量
- [ ] 数值列自动转型，非法值转 NaN 不报错

---

### 模块 6｜特征工程与建模

**功能**：构造派生特征、多目标加权、训练并交叉验证多模型。

**验收标准**
- [ ] 派生特征由领域包配置生成（雪茄示例：积温、温度范围、糖碱比、氮碱比、还原糖比例、质量指数）
- [ ] 多目标：每目标可设方向（正/负向）+ 权重，合成单一加权目标
- [ ] 支持至少 14 种回归模型（Ridge/Lasso/ElasticNet/SVR/KNN/DecisionTree/RandomForest/ExtraTrees/GradientBoosting/HistGradientBoosting/AdaBoost/Bagging/XGBoost/LightGBM），用户勾选
- [ ] 三种验证：GroupKFold / LeaveOneGroupOut / KFold，按分组列防数据泄漏
- [ ] 预处理管道：数值中位数填补+标准化，类别众数填补+one-hot
- [ ] 输出 R²/RMSE/MAE、逐折指标、CV 预测、特征重要性，自动选最优
- [ ] 有效样本不足（阈值可配，默认<8）时拒绝训练并提示
- [ ] 单模型失败不影响其他模型，失败单独记录
- [ ] 超参数可界面调整并写入建模配置 JSON
- [ ] （科学性修复）多目标标准化在训练折内进行，避免用测试折统计量造成泄漏
- [ ] （扩展）预留分类任务能力（部分生物场景目标是类别，如成熟等级/优劣判定）

---

### 模块 7｜优化与推荐

**功能**：搜索最优可控输入组合，做适用域与不确定性分析，生成验证实验方案。

**验收标准**
- [ ] 搜索变量与范围**来自领域包的"优化变量"定义**（不写死温度/含水率/时间）
- [ ] 网格搜索 + 按预测值排序输出 Top-N
- [ ] 适用域分析：标记超出训练范围的推荐点
- [ ] 多模型集成给出预测均值与标准差（不确定性），对超域点惩罚
- [ ] 支持 maximize / minimize
- [ ] 生成验证实验方案（对照组 + 推荐组，采样计划与检测项，模板来自领域包）
- [ ] 推荐结果落盘 CSV
- [ ] （扩展）预留更优搜索算法（贝叶斯优化/随机搜索）以应对高维输入

---

### 模块 8｜报告导出

**功能**：一键生成结构完整的学术报告（Word，预留 PDF）。

**验收标准**
- [ ] 报告章节结构与文案来自领域包报告模板
- [ ] 通用章节：摘要、目标、开源软件、流程图、（可选PRISMA）、数据提取与追溯、字段/单位/质量、预处理与派生特征、建模设计、模型结果与公式、优化推荐、验证方案、局限性、可复现性、发表建议
- [ ] 中文正常显示（宋体 + 西文 Times New Roman），支持中英文模板
- [ ] 自动配图：流程图、（PRISMA）、模型对比图、各模型预测散点+残差图
- [ ] 只写入用户实际选择且成功训练的模型
- [ ] 每个模型附公式、方法说明、可行性说明
- [ ] 缺步骤数据时对应章节占位说明，不崩溃

---

### 模块 9｜项目/运行管理与可追溯（横切）

**验收标准**
- [ ] 支持"项目"概念：一个项目绑定一个领域包，包含多次运行
- [ ] 每次运行保存：输入文件 hash、逐页文本、候选记录、curated 数据集、筛选决定、单位换算日志、质量报告、模型指标/逐折/CV预测、推荐结果、报告配图、模型元数据、随机种子、使用的领域包快照
- [ ] lineage 完整记录输入与配置（含领域包版本）
- [ ] 相同输入+配置+种子 → 结果可复现

## 8. 数据契约与领域配置规范（给架构侧的接口约定）

领域包建议采用声明式配置（YAML/JSON），至少包含以下段落：

```yaml
domain: cigar_fermentation        # 领域标识
display_name: 雪茄烟叶发酵
language: zh
fields:                           # 字段 Schema
  - name: temperature_mean
    label: 发酵温度
    type: numeric
    unit: degC
    role: [input]                 # input / target / trace / meta
    valid_range: [15, 70]
    extract_keywords: ["发酵温度","温度","temperature"]
  - name: sensory_total_score
    label: 感官总分
    type: numeric
    role: [target]
    direction: positive           # 目标方向
units:                            # 单位换算
  - field_pattern: temperature*
    conversions: [{from: F, to: degC, formula: "(x-32)*5/9"}]
derived_features:                 # 派生特征
  - name: degree_days
    formula: "temperature_mean * fermentation_days"
quality:
  sections: {traceability: [...], process: [...], outcome: [...]}
optimization:
  search_variables: [temperature_mean, leaf_moisture, fermentation_days]
report:
  template: cigar_report_zh
  sections: [...]
```

> 以上仅为需求层面的字段清单，具体格式由架构侧最终确定；核心要求是**领域知识全部可声明、内核零硬编码**。

## 9. 领域适配示例（验证"通用性"）

平台交付时至少证明两个领域可用：

1. **雪茄烟叶发酵包**（原型迁移，回归验证功能不丢失）
2. **另一个生物领域包**（如食品发酵或微生物发酵，证明换配置即可换领域）

验收时用这两个包分别跑通"上传→抽取→建模→推荐→报告"全流程。

## 10. 总体验收标准

平台整体验收（Definition of Done）：

- [ ] **通用性**：新增一个领域包（不改内核代码）即可服务新生物领域，且至少 2 个领域包跑通全流程
- [ ] **完整链路**：9 大模块全部可用，数据在模块间正确流转
- [ ] **可追溯**：任一数值可回溯来源；lineage 完整
- [ ] **可复现**：相同输入+配置+种子结果一致
- [ ] **科学严谨**：分组验证防泄漏；多目标标准化无泄漏；置信度定义清晰不误导
- [ ] **鲁棒**：缺数据/异常输入有友好提示，不崩溃
- [ ] **报告**：能导出结构完整、中文正常、图文齐全的学术报告
- [ ] **可维护**：内核/领域分离，模块解耦，有基础测试
- [ ] **文档**：含使用说明 + 领域包编写指南

## 13. 风险与对策

| 风险 | 对策 |
|------|------|
| 过度通用导致复杂度失控 | 先支撑 2 个领域，配置规范"够用即可"，避免一次性做全 |
| PDF 抽取准确率不稳定 | 明确"候选+人工复核"定位；优先规则+单位约束，后续可接 LLM/表格抽取 |
| 小样本导致模型不可靠 | 保留样本量下限；强制分组验证；报告适用域与不确定性 |
| 数据泄漏影响科学性 | 标准化/特征工程纳入 CV 折内；分组验证防同源泄漏 |
| 不同领域单位/字段差异大 | 全部交给领域包声明；内核不假设任何领域字段 |
| 非技术用户上手难 | 默认参数+向导式流程+示例包；高级参数折叠 |

## 14. 术语表

- **领域包（Domain Pack）**：描述某生物领域字段、抽取、单位、质量、建模、报告规则的配置集合。
- **可追溯（Traceability）**：每条数据可回溯到来源文件、页码、原文。
- **适用域（Applicability Domain）**：推荐点是否落在训练数据覆盖范围内。
- **PRISMA**：系统综述文献筛选流程规范。
- **分组验证**：按 study/来源分组做交叉验证，防止同源数据泄漏。
