## 0. 一页读懂：客户到底要什么

客户已有一个**写死"雪茄发酵"领域知识**的 Streamlit 单机原型，跑通了这条科研方法链路：

```
文献/数据上传 → 智能抽取 → 文献筛选(PRISMA) → 单位换算与质量评分
   → 人工校对 → 特征工程与建模 → 工艺参数优化推荐 → 学术 Word 报告
```

原型能跑通，**只因为喂进去的正好是雪茄文献、撞上了写死在 `KEYWORD_MAP` 里的雪茄关键词**（33 个字段的中英正则全在代码里）。它没有任何"发现关键词"的能力——换成酱油/酶工程文献就一个字段都抽不到。

### 基本盘（本方案的核心决策）：字段/关键词由系统自动发现，用户零配置

客户真实场景是"**用户绝对不会去配置任何东西，只会上传手里的文献然后要结果**"。因此本方案**不采用"用户配置领域包"的思路**，而是：

```
用户上传文献
  → 系统解析正文/表格/图片(OCR)
  → 【自动发现】从这批文献里统计高频词 + 同义词聚类 + 大模型语义归类
       → 产出"这批文献里到底有哪些字段/指标"（随文献而定，非写死）
  → 用户一键确认/微调候选字段（勾选、改名、合并；不是配置，是审核）
  → 定向抽取数值（回原文，绑定证据/页码/坐标）
  → 质控 → 校对 → 数据集 CSV → 建模 → 优化推荐 → Word 报告
```

**关键点**：
- **字段来自文献本身**，所以换任何生物领域都无需改代码、无需用户配置——天然通用。
- **大模型（DeepSeek，已配密钥）只做辅助**：语义归类、同义词建议、字段命名；**数值必须回原文校验 + 人工确认**，不允许大模型编数。
- 用户操作 = "上传 → 瞄一眼候选字段点确认 → 看结果"，比原型更省事（原型连字段都不能改）。

### "领域包"退居为可选的"辅助召回词库"，不是用户必须配的东西

- 我们可**内置一份广谱生物学关键词库**（含雪茄那套）作为"起跑加速"，提高首轮发现的召回率；
- 但它只是**加分项**，不存在也能靠自动发现工作；用户永远不需要碰它。

## 2. 自动字段发现机制（基本盘核心）

### 2.1 目标

用户上传一批文献后，系统**自动**回答"这批文献里到底有哪些可建模的字段/指标"，用户只需审核确认，不需要预先知道或配置任何字段。

### 2.2 发现流程（后端，异步任务 `discover_fields`）

```
1. 解析产物（正文块 + 表格单元格 + 图注 + OCR 文本）作为语料
2. 候选抽取：
   - 数值型：识别"名词短语 + 数字 + 单位"模式（如"发酵温度 45℃"、"盐度 8%"）
   - 表头型：表格表头单元格本身就是字段候选
   - 术语型：高频名词短语（词频/文档频统计）
3. 归并：
   - 规则同义聚类（RapidFuzz 归一化文本，复用 main 已有 synonym 逻辑）
   - 大模型语义归类：把同义/中英不同写法归到一个标准字段，给出建议标准名 + 类别
     （工艺参数/化学指标/感官评价/微生物/元数据…类别也由模型建议，非写死）
4. 打分：每个候选字段给 出现文献数 / 出现次数 / 置信度 / 示例原句 / 建议单位
5. 落库为"候选字段"（复用 terms / term_categories / field_schemas 结构）
```

### 2.3 用户确认（前端，轻量审核而非配置）

- 一张候选字段表：勾选保留、改标准名、合并同义、设角色（输入/目标/追溯/元数据）、设单位方向。
- 一键"全部确认" → 生成该项目的**字段方案（field_schema）** → 进入定向抽取。
- 这一步是"审核"，不是"配置"：默认全选即可跑，进阶用户才细调。

### 2.4 大模型使用边界（对齐客户需求书 §13.5）

- 允许：语义归类、同义词合并建议、字段命名、类别建议。
- 禁止：编造数值。**所有数值必须回原文正则/表格抽取 + 值域校验 + 人工确认**，大模型不直接写入正式数据集。
- 本地部署/断网时可退化为纯规则+统计发现（不调外部 API）。

### 2.5 可复现与可追溯

- 发现结果、确认动作、所用大模型 `provider@model`、随机种子写入 lineage 与审计。
- 项目字段方案冻结后，历史运行指向冻结版本（复用现有冻结机制）。

### 2.6 "内置广谱词库"（可选加分项，非必需）

- 内置一份广谱生物学关键词库（含雪茄那套）**仅用于提升首轮召回**：发现时作为先验提示，命中即加分。
- 不存在也能靠 2.2 的统计+大模型发现工作；用户永远不需要维护它。

## 3. 分阶段落地计划（fr-v1）

> 原则：先打通"自动发现→抽取→数据集"主链，再补 ML/优化/报告；每阶段"后端 + 前端 + 测试"闭环；用雪茄真实文献作端到端回归基准。

### 阶段 A｜自动字段发现闭环（P0，最高优先，基本盘）✅ 已完成
- ✅ A1 后端：新增 `discover_fields` 任务（全项目文献，不依赖 search_run），实现候选抽取（数值模式"短语+数值+单位" + 表头单元格 + 高频短语）与打分，落库为候选字段（terms status=candidate + 别名 + 富元数据）。
- ✅ A2 大模型接入：DeepSeek 语义归类/同义合并/命名建议（走 external_calls 记账；无 key/断网退化为纯规则 + rapidfuzz 同义聚类）。数值绝不由模型生成。
- ✅ A3 一键确认 → `POST /field-schemas/from-candidates` 生成项目字段方案（复用 field_schemas / field_definitions，回填 source_term_id 与 extraction_config.aliases，供定向抽取消费）。
- ✅ A4 前端：术语页"自动发现字段"面板 → 候选字段审核卡片（勾选/全选/清空、类型/角色/类别/单位标签、文档数/出现次数/置信度/命中写法/示例原句）→ 一键确认生成字段方案。
- A5（可选）内置广谱关键词库作为召回先验 — 暂缓（作为加分项，后续按需补）。
- **接口**：`POST /projects/{id}/field-discovery`、`GET /projects/{id}/field-candidates`、`POST /projects/{id}/field-schemas/from-candidates`。
- **验收**：上传一批**任意生物领域**文献 → 系统给出候选字段清单 → 用户一键确认进入抽取，全程零配置。（待阶段 F 用雪茄+非雪茄文献做端到端回归）

### 阶段 B｜定向抽取 / 单位 / 质量（P0）🟡 进行中
- ✅ B1 定向抽取按"已确认字段"回原文抽数值（正文 + 规整表格），绑定证据/页码/坐标（已在 main 具备，A3 已把发现字段回填 source_term + aliases 供其消费）。
- ✅ B2 值域约束：抽取时按字段 `validation_rules` 的 min/max 校验数值，超范围者置信度下调并自动标"存疑"，元数据记录 `quality.range_violation`（修复 pH 误撞/字段串扰）。
- ✅ B3 单位换算 + 冲突：已有等价换算与 `conversion_records`；质量报告聚合换算状态与单位冲突字段。
- ✅ B4 **质量评分引擎**：`GET /extraction-runs/{id}/quality-report` —— 分区完整度 + 合理范围（有规则用规则、无规则用数据驱动 IQR 离群检测）+ 单位冲突 + 换算日志，产出 0–100 综合分与逐字段明细。
- ✅ B5 前端：抽取工作台新增"质量报告"面板（综合分、完整度/合理范围进度条、超范围计数、单位冲突标记、换算统计）；记录审核与 CSV 导出已具备。
- **接口**：`GET /projects/{id}/extraction-runs/{runId}/quality-report`。
- **验收**：确认字段后能抽出带证据数值、成表、打质量分、导出 CSV；换领域文献同样可用（待阶段 F 端到端回归）。

### 阶段 C｜建模通用化与短板补齐（P1）
- ✅ C1 模型注册表补齐至 **16 种回归 / 13 种分类**（新增 Lasso/ElasticNet/KNN/DecisionTree/ExtraTrees/HistGradientBoosting/AdaBoost/Bagging，LightGBM 可选，缺库自动跳过）。
- ✅ C2 **三种交叉验证主策略**：GroupKFold / LeaveOneGroupOut / KFold（`cv_strategy` 可选），按来源文献分组防泄漏；输出**逐折指标**（`cv_fold_N` 落 `ml_metrics`）+ **CV 预测落库**（`split_name=cross_validation`）；分组不足时**显式降级并告警**（`cv_warnings`），非静默。
- ✅ C3（泄漏核验）标准化置于 sklearn Pipeline 内，网格搜索与逐折评估**均折内拟合**，天然无全量泄漏；CV 在增强前的原始训练集上进行以保持样本-行映射。**多目标加权合成**待续（见下方"待续"）。
- C4 派生特征引擎：内置常见派生（比值/乘积/积温类）自动候选 + 用户可增删；无需用户写公式。
- C5 单模型失败隔离、样本量下限可配（默认 8）、超参前端可调、随机种子全局固定。
- ✅ C6 前端 ML 面板：模型勾选（含新增算法中文标签）、**CV 策略下拉**。目标权重/超参微调待续。
- **验收**：14+ 模型可选（后端全绿 + 真实 PostgreSQL 端到端训练通过）、3 CV 可选、无数据泄漏。
- **待续**：C3 多目标加权（多目标方向+权重、折内归一化合成）、C4 派生特征、C5 超参前端微调、C6 目标权重面板。

### 阶段 D｜优化与推荐补齐（P1）
- D1 网格搜索（除现有随机搜索）；搜索变量默认取"输入类字段"，范围默认取训练数据范围，用户可调。
- D2 适用域分析：标记超训练范围点并惩罚；多模型集成均值+标准差。
- D3 验证实验方案生成（对照组 + Top-N 推荐组）。
- D4 推荐结果 CSV/Excel 落盘；maximize/minimize 路径统一。
- **验收**：输出 Top-N 候选 + 适用域 + 不确定性 + 验证方案。

### 阶段 E｜PRISMA 与报告模板化（P2）
- E1 PRISMA 模块（表 + API + UI + PRISMA 计数图），**项目级可开关/隐藏**（纯实验数据项目可跳过）。
- E2 报告模板化：补齐客户章节清单（摘要/目标/开源软件/流程图/PRISMA/数据提取与追溯/字段单位质量/预处理与派生特征/建模设计/模型结果与公式/优化推荐/验证方案/局限性/可复现性/发表建议）；缺步骤章节占位不崩溃；仅写用户选中且成功的模型；中文字体正常。
- **验收**：报告章节完整、中文正常、图文齐全、缺数据不崩。

### 阶段 F｜跨领域验证 + 收尾（验收关键）
- F1 用**雪茄真实文献**跑通"上传→自动发现→确认→抽取→建模→推荐→报告"全流程（首个端到端 demo）。
- F2 再用**一个非雪茄生物领域文献**跑通全流程，证明"零配置换领域"（客户验收硬指标）。
- F3 接入补齐：CSV 支持、默认 1GB 上限、run 级 lineage 聚合、依赖锁定。
- F4 文档：使用说明（面向零基础用户）+ 自动发现机制说明。
- **验收**：客户总体验收标准（通用性/完整链路/可追溯/可复现/科学严谨/鲁棒/报告/可维护/文档）全绿。
